# Tabi-Pay 開発ログ

## 技術スタック

| 項目 | 採用技術 | 理由 |
|---|---|---|
| フロント | Next.js 15 (App Router / TypeScript) | LIFFと相性良好、API Routesで完結 |
| スタイル | Tailwind CSS | 高速UI構築 |
| DB | Supabase (PostgreSQL) | 無料枠・RLSでアクセス制御 |
| 画像保存 | Supabase Storage | 無料1GB、DBと一体管理 |
| ホスティング | Vercel | Next.js公式、HTTPS自動、無料 |
| LINE連携 | LIFF SDK v2 | グループID自動取得でメンバー同期 |
| 通知 | LINE Messaging API (Push) | 月200通無料枠 |
| Cron | Vercel Cron Jobs | 毎晩22:00バッチ、無料2本まで |

## コスト試算

すべて無料枠で運用可能（3人グループ想定）

### [T-001] create-next-app がフォルダ名の大文字でエラー

- **発生日**: 2026-05-21
- **症状**: `create-next-app . ` 実行時に `name can no longer contain capital letters` エラー
- **原因**: npm のパッケージ名規則で大文字不可。フォルダ名 `AccountingBook` がそのままパッケージ名になるため
- **対処**: `create-next-app` を使わず、`package.json`（name: `tabi-pay`）など設定ファイルを手動作成し `npm install` で解決

### [T-004] LINEグループから開いてもずっと起動中のまま

- **発生日**: 2026-05-22
- **症状**: `miniapp.line.me` のURLをLINEグループで開いても「起動中...」のまま無限ローディング
- **原因**: `LiffProvider.tsx` で `next/script` の `onLoad` → `setState` → `useEffect` の間接的な連鎖が不安定で、`init()` が呼ばれないケースがあった。また `liff.init()` がタイムアウトしてもエラーを表示しなかった
- **対処 v1**:
  - `onLoad` から直接 `init()` を呼ぶよう変更（`scriptLoaded` state を廃止）
  - `liff.init()` に10秒のタイムアウトを追加（`Promise.race`）
  - `onError` ハンドラを追加してSDK読み込み失敗を検知
  - `useRef` で二重初期化を防止
- **追加対処 v2**: 7ステップのデバッグ表示を追加
- **追加対処 v3（ループ継続のため）**: `liff.login()` の自動呼び出しをやめ、「LINEでログイン」ボタン方式に変更。LINE内ブラウザで `liff.login()` を自動呼び出しするとページ再読み込みが繰り返されるループが発生していた。デバッグログを画面上に常時表示し、どのステップで止まっているか目視できるようにした

### [T-003] cron route で Supabase 部分 select の型不一致

- **発生日**: 2026-05-22
- **症状**: `npx tsc --noEmit` で `Expense` 型と Supabase の部分 select 結果の型が不一致
- **原因**: `select("amount, payment_type")` の戻り値が `{ amount: any; payment_type: any; }[]` になり、`Expense` 型を期待するコールバックと不一致
- **対処**: ローカルに `type PartialExpense` を定義してキャストすることで解決

---

### [T-002] Next.js 15.3.2 のセキュリティ脆弱性

- **発生日**: 2026-05-21
- **症状**: `npm install` 後に CVE-2025-66478 の警告（critical）
- **対処**: `npm install next@latest` で最新版（16.x）に更新
- **残存**: Next.js 内部バンドルの `postcss < 8.5.10`（moderate）が残存。`--force` で修正すると Next.js が 9.x にダウングレードされるため意図的に放置。CSS サーバーサイド stringify 時のみ発生する XSS であり、本アプリの用途（LIFF内クライアントレンダリング）では影響なし。Next.js 側のパッチ待ち
